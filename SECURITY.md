# Security Policy

## Supported version

Security fixes are applied to the latest code on the `main` branch.

## Reporting a vulnerability

Please do not open a public issue for a vulnerability that could expose financial records, invoice attachments, or backup data. Email **tolstoyjustin@gmail.com** with a description, reproduction steps, affected version, and any suggested mitigation.

The application is local-first, but exported reports and `.betbackup.zip` archives can contain sensitive financial information. Users should keep those files in protected storage and share them only with trusted recipients.
